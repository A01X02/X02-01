## 第二步：获取连接密钥（3 个值需要复制下来）

登录 [腾讯云 CloudBase 控制台](https://console.cloud.tencent.com/tcb/env/index)：

1. **环境 ID（ENV_ID）**：`x02-01` ← 你已经有了
2. **Publishable Key（等价于 Supabase 的 anon key，前端用）**：
   - 控制台 → 点进 `x02-01` 环境 → 左侧「**设置**」→「**API Key 配置**」
   - 复制「**Publishable Key**」（一串字母数字）
3. **服务端 API Key（等价于 Supabase 的 service_role key，仅服务端用）**：
   - 同一页面 → 复制「**服务端 Key**」（Server API Key）

> ⚠️ 服务端 Key 不要泄露给任何人！它相当于数据库管理员密码（BYPASS RLS）。
> 它要填到环境变量 `CLOUDBASE_APIKEY` —— 适配层会自动读取，无需在前端出现。

把这 3 个值记在安全的地方，下一步要填到代码里。

## 第五步：配置环境变量

在 CloudBase 云托管中设置环境变量：

1. 控制台 → `x02-01` → 左侧「**云托管**」→「**环境变量**」
2. 逐项添加（参考 `env.domestic.example`）：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `NEXT_PUBLIC_DEPLOY_ENV` | `domestic` | 触发国内模式 |
| `NEXT_PUBLIC_CLOUDBASE_ENV_ID` | `x02-01` | 你的环境 ID |
| `NEXT_PUBLIC_CLOUDBASE_ACCESS_KEY` | （第二步复制的 Publishable Key） | 前端可见 |
| `CLOUDBASE_APIKEY` | （第二步复制的**服务端 Key**） | 仅服务端，BYPASS RLS |
| `NEXT_PUBLIC_CLOUDBASE_REGION` | `ap-shanghai` | x02-01 所在地域 |
| `DOUBAO_API_KEY` | （和境外版一样） | 豆包 API Key |
| `DOUBAO_API_ENDPOINT` | （和境外版一样） | 豆包地址 |
| `DOUBAO_MODEL_ID` | （和境外版一样） | 豆包模型 ID |
| `VOLC_TTS_*` | （和境外版一样，4 个变量） | 火山语音 |

> ⚠️ 豆包和火山引擎的变量直接从你 Vercel 的环境变量复制过来就行——它们本来就是国内服务，不需要换。

## 常见问题

### Q: 免费版 6 个月后怎么办？
A: 到期后自动转按量付费或续兑换码。你也可以升级到个人版 ¥19.90/月。对于自用量，免费额度内的资源点（3000点/月）通常够用。

### Q: 和境外版的代码怎么共存？
A: 完全独立。代码通过 `NEXT_PUBLIC_DEPLOY_ENV=domestic` 自动切换：
- 不设 / 设成其他值 → 用 Supabase（境外 Vercel）
- 设为 `domestic` → 用 CloudBase（国内）

同一份代码，两套 env，互不影响。

### Q: 数据库操作真的不用改代码？
A: 对！CloudBase PG 基于 PostgREST，`.from().select()/.insert()` 这些链式调用与 Supabase 完全一致。我已经写了适配层（`src/lib/cloudbase-client.ts`），Auth 的参数差异也在适配层内部处理了。业务代码零改动。
