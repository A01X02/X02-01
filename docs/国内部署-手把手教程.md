## 第二步：获取连接密钥（3 个值需要复制下来）

登录 [腾讯云 CloudBase 控制台](https://console.cloud.tencent.com/tcb/env/index)：

1. **环境 ID（ENV_ID）**：`x02-01` ← 你已经有了
2. **Publishable Key（等价于 Supabase 的 anon key，前端用）**：
   - 控制台 → 点进 `x02-01` 环境 → 左侧「**设置**」→「**API Key 配置**」
   - 复制「**Publishable Key**」（一串字母数字）
3. **服务端 API Key（等价于 Supabase 的 service_role key，仅服务端用）**：
   - 同一页面 → 复制「**服务端 Key**」（Server API Key）

> ⚠️ 服务端 Key 不要泄露给任何人！它相当于数据库管理员密码（BYPASS RLS）。
> 它要填到环境变量 `CLOUDBASE_APIKEY` —— 适配层会自动读取，无需在前端出现。

把这 3 个值记在安全的地方，下一步要填到代码里。

## 第四步：建表（执行 SQL）

项目里准备了**两个**建表脚本，国内部署请用 **CloudBase 专用版**：

| 文件 | 用途 |
|------|------|
| `supabase-schema.cloudbase.sql` | ✅ **国内用这个**（已适配 CloudBase PG：免扩展 UUID、去掉 auth.users 外键、触发器隔离） |
| `supabase-schema.sql` | 海外 Supabase 原版（不要在国内跑） |

### 方式 A：用 CloudBase 控制台的 SQL 编辑器（推荐新手）

1. 控制台 → `x02-01` →「**数据库**」→「**PostgreSQL**」→「**SQL 编辑器**」
2. 打开项目里的 `supabase-schema.cloudbase.sql`
3. **逐块粘贴执行**：脚本已用「【第 N 段】」拆成独立块，每块以分号结尾，建议一块一块跑（CloudBase 编辑器通常一次跑一条语句）
4. 跑到【第 11 段·可选】时：
   - 如果报 `auth.users 不存在` / 没有权限 → **直接跳过**，不影响使用
   - 跳过后的影响：新用户首次登录后，去「设置」改一次昵称即可自动生成 profile 行

### 方式 B：用 ExecutePGSql 云 API（适合批量/自动化）

如果你会一点 Python，可以用腾讯云 SDK 把整份脚本一次性灌进去（需要你的腾讯云 SecretId/SecretKey，不是 App 的 API Key）：

```bash

```

`profiles`、`messages`、`conversations`、`memories`、`moments`、`comments`、`likes`、`voice_profiles`、`user_settings`、`chat_themes`

## 第五步：配置环境变量

在 CloudBase 云托管中设置环境变量。**注意区分「构建时」和「运行时」**：

- **构建时环境变量**（Next.js 构建时烧进前端，必须填，否则白屏）：
  CloudBase 控制台 → `x02-01` →「云托管」→「**构建**」相关设置里填；
  或直接用 Dockerfile 里已写死的默认值（ENV_ID=x02-01、REGION=ap-shanghai、DEPLOY_ENV=domestic）。
  **你只需额外补 2 个构建变量**：
  | 变量名 | 值 |
  |--------|-----|
  | `NEXT_PUBLIC_CLOUDBASE_ACCESS_KEY` | （第二步复制的 Publishable Key） |
  | `NEXT_PUBLIC_SITE_URL` | `https://aeromaxx.ltd`（或 CloudBase 分配的临时域名） |

- **运行时环境变量**（容器启动后读取，在「云托管 → 环境变量」填）：
  | 变量名 | 值 | 说明 |
  |--------|-----|------|
  | `CLOUDBASE_APIKEY` | （第二步复制的**服务端 Key**） | 仅服务端，BYPASS RLS |
  | `DOUBAO_API_KEY` | （和境外版一样） | 豆包 API Key |
  | `DOUBAO_API_ENDPOINT` | （和境外版一样） | 豆包地址 |
  | `DOUBAO_MODEL_ID` | （和境外版一样） | 豆包模型 ID |
  | `VOLC_TTS_*` | （和境外版一样，4 个变量） | 火山语音 |

> ⚠️ 豆包和火山引擎的变量直接从你 Vercel 的环境变量复制过来就行——它们本来就是国内服务，不需要换。
> ⚠️ `NEXT_PUBLIC_` 开头的变量若只在「运行时」填、没在「构建时」填，前端会拿到 undefined → 白屏。务必按上面区分填写。

## 第六步：部署应用到 CloudBase

### 方法 A：通过 CloudBase 控制台上传（推荐，免 GitHub）

1. 在本地打包项目：
   ```bash
   cd H:\Games\AI_Project_chatbot
   npm run build
   # 或者用 Docker（项目根目录已有 Dockerfile）
   docker build -t ai-chat-app .
   ```

2. 控制台 → `x02-01` →「**云托管**」→「**新建版本**」
3. 上传打包产物或推送 Docker 镜像
4. CloudBase 会自动分配一个访问域名（类似 `x02-01.ap-shanghai.app.tcloudbase.com`）

### 方法 B：通过 GitHub 自动部署

1. 把代码 push 到 GitHub（你已经有了）
2. CloudBase 控制台 →「**持续部署**」→ 关联 GitHub 仓库
3. 配置构建命令和环境变量
4. 每次 push 自动触发部署

## 第七步：（可选）绑定你的域名 aeromaxx.ltd

如果你想用自己的域名访问：

1. **ICP 备案**（必须）：在腾讯云 →「备案管理」→ 提交 aeromaxx.ltd 的备案
   - 通常 1–2 周审核通过
   - `.ltd` 后缀可以备案（以注册商后台提示为准）
2. **添加自定义域名**：
   - CloudBase 控制台 →「**基础配置**」→「**域名管理**」→「**添加域名**」
   - 输入 `aeromaxx.ltd` 或 `app.aeromaxx.ltd`
3. **DNS 解析**：
   - 到你的域名注册商（你买 aeromaxx.ltd 的地方）
   - 添加 CNAME 记录指向 CloudBase 分配的域名

> ⚠️ **没备案前也能用**：先用 CloudBase 分配的免费域名测试，备案期间不影响使用。

## 常见问题

### Q: 免费版 6 个月后怎么办？
A: 到期后自动转按量付费或续兑换码。你也可以升级到个人版 ¥19.90/月。对于自用量，免费额度内的资源点（3000点/月）通常够用。

### Q: 和境外版的代码怎么共存？
A: 完全独立。代码通过 `NEXT_PUBLIC_DEPLOY_ENV=domestic` 自动切换：
- 不设 / 设成其他值 → 用 Supabase（境外 Vercel）
- 设为 `domestic` → 用 CloudBase（国内）

同一份代码，两套 env，互不影响。

### Q: 数据库操作真的不用改代码？
A: 对！CloudBase PG 基于 PostgREST，`.from().select()/.insert()` 这些链式调用与 Supabase 完全一致。我已经写了适配层（`src/lib/cloudbase-client.ts`），Auth 的参数差异也在适配层内部处理了。业务代码零改动。
