# 国内部署 · 手把手教程（自建 Supabase + CloudBase）

> 写给米米：不改动境外那套（Vercel + Supabase），在国内另搭一套免翻墙可用的镜像。
> 全程照着做即可，遇到任何一步卡住，把截图发我，我帮你接着排。
>
> **核心记住一句话**：让国内能用，要换的是 ①数据库 Supabase ②部署平台 Vercel。
> GitHub 不用换（它只是存代码，用户从不访问）。


## 第一部分：搭建国内 Supabase（数据库）

### 步骤 1 · 买一台轻量应用服务器

1. 打开 [腾讯云轻量应用服务器](https://cloud.tencent.com/product/lighthouse)，登录 + 实名
2. 点「立即购买」，配置这样选：
   - **地域**：选离你近的（如上海/广州/北京）
   - **镜像**：应用镜像里找 **Docker**，或系统镜像选 **Ubuntu 22.04**
   - **套餐**：⚠️ **至少 2核4G**（Supabase 全家桶较吃内存，2核2G 会很吃力，预算够就选 4核8G）
   - **时长**：先买 1 个月试水
3. 购买后进入「服务器详情」，记下 **公网 IP**（后面处处要用）
4. 在「防火墙」里放行这些端口：`22`(SSH)、`80`、`443`、`8000`(Supabase API)、`3000`(Studio 管理台)

### 步骤 2 · 连上服务器

- Windows 用户最简单：在轻量服务器控制台点「**登录**」→ 网页版终端直接进（不用装软件）
- 或用你电脑的 PowerShell：`ssh ubuntu@你的公网IP`（首次会让你设密码）

### 步骤 3 · 装 Docker（如果买的是纯 Ubuntu 镜像）

在服务器终端里逐行执行：

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 一键装 Docker
curl -fsSL https://get.docker.com | sudo bash

# 让 docker 命令免 sudo（可选）
sudo usermod -aG docker $USER && newgrp docker

# 验证
docker --version
```

> 如果买的是 Docker 应用镜像，跳过这步。

### 步骤 4 · 部署 Supabase

```bash
# 下载 Supabase 官方自托管包
git clone --depth 1 https://github.com/supabase/supabase
cd supabase/docker

# 复制配置模板
cp .env.example .env
```

**关键：改 `.env` 里的密码和密钥**（不改的话是公开默认值，极不安全）：

```bash
nano .env    # 打开编辑，改完 Ctrl+O 回车保存，Ctrl+X 退出
```

至少改这几项（随便设成你自己的强密码/随机串）：
- `POSTGRES_PASSWORD=` 设一个强密码
- `JWT_SECRET=` 一串 40 位以上随机字符
- `ANON_KEY=` 和 `SERVICE_ROLE_KEY=` ← **这两个后面 APP 要用**，官方 .env 里有默认值，生产建议按[官方工具](https://supabase.com/docs/guides/self-hosting/docker#generate-api-keys)重新生成
- `DASHBOARD_USERNAME=` / `DASHBOARD_PASSWORD=` 管理台登录用

启动：

```bash
docker compose up -d      # 第一次会拉镜像，等几分钟
docker compose ps         # 全部 State=running 就成功了
```

启动成功后你就有了：
- **API 地址**：`http://你的公网IP:8000` ← 这是 `NEXT_PUBLIC_SUPABASE_URL` 的雏形（下面要升级成 HTTPS）
- **管理台 Studio**：浏览器打开 `http://你的公网IP:3000`，用上面设的账号密码登录
- **anon key / service_role key**：就是你 `.env` 里那两个

### 步骤 5 · ⚠️ 最重要的坑：给 Supabase 配 HTTPS

**为什么必须做**：你的 APP 会跑在 CloudBase 的 `https://` 域名下。浏览器有条铁律——**HTTPS 页面不允许调用 HTTP 接口**（叫「混合内容拦截」）。如果 Supabase 还是 `http://IP:8000`，那么在线上聊天/登录会被浏览器直接拦掉，和现在被墙的表现很像。

所以 Supabase 必须有自己的 **HTTPS 域名**。最省事的方案是用 **Caddy**（自动申请免费证书）：

1. **准备域名**：买个域名（如 `mimibot.com`），加一条解析记录，比如 `api.mimibot.com` → 指向你服务器公网 IP
2. **备案**：域名指向国内服务器，必须备案（腾讯云控制台「备案」入口提交，等 1-2 周）。⚠️ 建议**今天就先提交备案**，别等其他都弄好了卡在这
3. 备案通过后，在服务器上装 Caddy 做反向代理：

```bash
# 装 Caddy
sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt update && sudo apt install -y caddy

# 配置反代（自动 HTTPS）
sudo nano /etc/caddy/Caddyfile
```

`Caddyfile` 内容改成：

```
api.mimibot.com {
    reverse_proxy localhost:8000
}
```

```bash
sudo systemctl reload caddy    # Caddy 会自动申请 HTTPS 证书
```

现在你的 Supabase 正式地址就是 **`https://api.mimibot.com`** —— 这就是要填给 APP 的 `NEXT_PUBLIC_SUPABASE_URL`。

> 💡 不想弄域名+备案？那就得选之前说的 **B 方案（Sealos）**，它自带 HTTPS 域名，省掉这一整段。你要是觉得这段太折腾，跟我说，我给你出 Sealos 版教程。

## 第二部分：建表 + 把老数据搬过来

### 步骤 6 · 建表

项目根目录有现成的建表脚本 `supabase-schema.sql`（277 行，包含 profiles / 聊天 / 朋友圈 / 记忆等所有表）。

1. 浏览器打开你的国内 Studio：`http://你的公网IP:3000`
2. 左侧点 **SQL Editor** → New query
3. 把 `supabase-schema.sql` 全文复制粘贴进去 → 点 **Run**
4. 脚本是「幂等」的（可重复执行不报错），跑完所有表就建好了

### 步骤 7 · 迁移老数据（把海外库的记忆/聊天搬到国内）⭐

你现在海外 Supabase 里已经有数据（尤其是你调教了很久的**记忆**）。别丢，搬过来：

**方法：用 Supabase 官方的导出/导入**

1. 需要你能连上海外 Supabase（这一步要翻墙，因为要从海外库导出）
2. 在你本地电脑装 PostgreSQL 客户端工具（含 `pg_dump` / `psql`）
3. 从海外库导出数据：

```bash
# <海外连接串> 在海外 Supabase 后台 → Settings → Database → Connection string 里找
pg_dump "海外连接串" --data-only --schema=public -f backup.sql
```

4. 导入国内库：

```bash
psql "postgresql://postgres:你设的密码@你的公网IP:5432/postgres" -f backup.sql
```

> ⚠️ `auth.users`（登录账号）表比较特殊，如果搬完发现登录不上，把情况发我，我给你单独处理账号迁移（或者国内库重新注册一个账号也行，反正主要是自己用）。
> 数据量不大的话，其实**在国内重新注册 + 把记忆重新填一遍**也很快，你自己权衡。

## 第三部分：把 APP 部署到 CloudBase

### 步骤 8 · 开通 CloudBase

1. 打开 [CloudBase 控制台](https://console.cloud.tencent.com/tcb)
2. 首次进入会让你**实名 + 开通**（免费，跟着走）
3. 点「新建环境」，计费选「**按量付费**」（个人自用免费额度基本够），记下**环境 ID**

### 步骤 9 · 部署（控制台方式，最简单）

1. CloudBase 控制台 → 左侧「**云托管**」→ 新建服务
2. 代码来源：
   - 能翻墙 push 的话，关联 **GitHub 仓库**
   - 不方便的话，选「**本地代码上传**」，把项目打包成 zip 传上去（不经过 GitHub，正好绕开你说的翻墙问题）
3. 构建方式选 **Docker**（用项目根目录的 `Dockerfile`，我已经准备好了）
4. **填环境变量**（这步最关键，见下面步骤 10）
5. 点部署，等构建完成
6. 完成后 CloudBase 分配一个 `xxx.ap-shanghai.app.tcloudbase.com` 之类的**免备案子域名**，直接就能在国内手机打开

### 步骤 10 · 环境变量对照表（照着填）

在 CloudBase 云托管的「环境变量」里，填这些：

| 变量名 | 填什么 | 说明 |
|---|---|---|
| `NEXT_PUBLIC_DEPLOY_ENV` | `domestic` | **必填**，触发国内构建模式 |
| `NEXT_PUBLIC_SUPABASE_URL` | `https://api.mimibot.com` | 你国内 Supabase 的 HTTPS 地址 |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 国内库的 anon key | 步骤 4 里的 |
| `SUPABASE_SERVICE_ROLE_KEY` | 国内库的 service_role key | 步骤 4 里的 |
| `DOUBAO_API_KEY` | 你的豆包 key | 和海外版一样，不用改 |
| `DOUBAO_API_ENDPOINT` | 豆包地址 | 一样 |
| `DOUBAO_MODEL_ID` | 豆包模型 ID | 一样 |
| `VOLC_TTS_APP_ID` 等 4 个 | 你的火山语音配置 | 一样 |

> ⚠️ `NEXT_PUBLIC_` 开头的变量要在「**构建环境变量**」和「**运行环境变量**」里**都填一遍**（这类变量会在构建时被写进前端代码）。

## 第四部分：验证 + 常见问题

### 步骤 11 · 验证

1. 手机（关掉翻墙）打开 CloudBase 给的子域名
2. 依次测：能注册/登录？能聊天回复？记忆能加载？语音能播？
3. 全绿就成功了 🎉，可以「添加到主屏幕」当 APP 用（PWA 已配好）

### 常见问题速查

| 现象 | 多半是 | 怎么办 |
|---|---|---|
| 界面能开，登录/聊天失败 | Supabase 地址还是 http，被混合内容拦 | 回步骤 5 配 HTTPS |
| 构建失败提示拉字体超时 | 没设 `NEXT_PUBLIC_DEPLOY_ENV=domestic` | 补上这个变量重新构建 |
| 图片/头像加载不出 | 图片域名没配对 | 确认 `NEXT_PUBLIC_SUPABASE_URL` 填对 |
| Supabase 容器起不来 | 服务器内存不够 | 升到 4核8G，或关掉不用的服务 |
| 域名打不开 | 备案没通过 / 解析没生效 | 等备案，或先用 IP+端口本地测 |
